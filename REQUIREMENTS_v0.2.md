# BPRecorder v0.2 需求文档

## v0.2 目标

模块化改造 + 多用户预留框架

---

## 一、模块化

### 1.1 识别模块

#### 布局调整
- 右上角显示当前识别方式（如"前端听写"）
- "听写"按钮改为"识别"按钮
- 描述改为："支持直接输入和{当前选择方式}是方式而不是这六个汉字"，并且把描述的位置改到识别区（输入区）之下一行，和标题行不能在同一行
- 配置区的最终选择项，传递到{当前选择方式},识别按钮通过函数调用关联到{当前选择方式}
- 长按按钮时禁止出现文字选择框

#### 识别按钮行为
- 语音识别：显示"识别"+"停止"两个状态，也就是有第一屏和第二屏
- 蓝牙/拍照：只有"识别"（一次性动作），也就是只有第一屏没有第二屏
- 但是不论是一屏还是两屏，都不能影响下面的提取按钮，提取按钮不管什么时候都是可以点击的，如果输入框是空内容，那么提取后生成的也是空内容
- 点击"识别" → 执行当前选中的识别方式

#### 识别配置方式
- **长按**“识别”按钮 → 进入识别配置区
- 可选识别方式：
  - 前端听写（当前实现，Apple/微软自动切换）
  - 云端听写（预留）
  - 蓝牙识别（预留）
  - 拍照识别（预留）

#### 识别配置区内容
- 当前方式选择
- 云端听写：API类型(飞书/通义/Whisper) + URL + Token
- 蓝牙：系统配对设备列表 + 选择
- 拍照：暂空

#### 提取配置方式和内容
- **长按**“提取”按钮 进入提取配置区
- 可选提取方式：
 - 正则，正则下面是正则公式内容，把目前代码中硬编码的正则公式放在这里，通过函数来调用，去掉硬编码方式
 - 模型，模型下面是两行内容，第一行是模型的url，第二行是token，如果该模型不需要token就不填写，第三行是提示词，如果选择了模型并把url/token/提示词都填写了，才能选择模型方式，并用函数或者其他方式传递到提取按钮，按提取按钮后调用这里选的提取方式并使用正则或者提示词
 - 当前只有正则一个选项

#### 模块化目标
- 新增识别方式 → 只需增加新模块代码 + 修改配置区
- 不改动主逻辑

---

### 1.2 提取模块

#### 配置方式
- **长按**"提取"按钮 → 进入提取配置区

#### 配置内容
- 可选提取方式：
  - 正则：正则公式内容可编辑，把目前代码中硬编码的正则公式放在这里，通过函数来调用
  - 模型：模型url + token（可选）+ 提示词，三者都填写后才能选择模型方式
- 当前只有正则一个选项可用

#### 提取按钮行为
- 点击"提取"按钮 → 调用配置中选择的提取方式（正则或模型）
- 提取结果放入待选区

#### 模块化目标
- 新增提取方式 → 只需增加新模块代码 + 修改配置区
- 不改动主逻辑


---

### 1.3 存储模块（记录区）

#### 当前硬编码位置
- 文件：index.html
- 代码：line 715 `const text = \`${item.time}---${item.high}/${item.low}(${item.diff})-${item.pulse}🩺\`;`
- 需改造为可配置的格式化字符串

#### 记录区格式
- 可自定义格式化字符串
- 示例：
  - `HH:MM---高压/低压(压差)-心率🩺` → `23:45---120/80(40)-60🩺`
  - `HH:MM---高压/低压-压差-心率🩺` → `23:45---120/80-40-60🩺`
  - `HH:MM---高压/低压心率🩺` → `23:45---120/8060🩺`
- 压差显示/隐藏、压差符号 `()` 或 `/` 都在格式字符串中体现

#### 备注处理
- 备注显示在记录区最下方一行
- 复制按钮只复制正式内容，不含备注

#### 配置方式
- **长按**记录区标题 → 进入记录配置区

#### 当前状态
- 刷新后记录消失（纯内存）
- 预留云端存储选项

#### 模块化目标
- 新增存储方式 → 只需增加新模块代码 + 修改配置区
- 不改动主逻辑

## 二、多用户系统

### 2.1 用户状态

#### 未注册/未登录
- 使用模块化后的基础功能
- 无云端存储

#### 已登录
- 可选择云端存储（预留）
- 数据存到云端

### 2.2 注册登录

#### 注册
- 用户名 + 密码 + 邮箱
- 密码存储需加密

#### 登录
- 用户名 + 密码

#### 密码找回
- 邮箱验证码方式
- 预留框架

### 2.3 配置文件
- 我们在第一个章节中提到的三种配置，分别是识别方式/提取逻辑/记录格式，需要写到配置文件中，单击识别按钮/提取按钮/待选区最下面的采纳按钮的时候，通过配置文件中的内容来调用相关的方式和内容
- 如果用户没有注册，应该用cokie来做标识，在当前目录下新建一个用户身份的目录，在这个目录下按不同cokie来保存不同的用户身份
- 如果是注册用户，就用用户名来做标识，在当前目录下的用户身份目录下，按注册的用户名生成目录名，在这个用户目录下，写入配置文件
- 所有用户的初始状态，也是未注册用户的第一次进入页面，和注册用户登录后第一次进入页面，此时采用默认配置文件，默认配置文件就是采用目前已经跑通的配置，识别方式是前端听写，提取方式是正则，正则公式是当前公式，格式化的格式是当前格式
- 既然有了配置文件，那么本着简单的原则，其他还有什么类似的内容，都可以写到配置文件中，总之达到的目标是不启用用户注册之前，也是按照cokie来区分的多用户状态

## v0.3 需求预览

### 提取模块增强

#### 纯数字拖动识别
- 适用场景：纯数字无前缀（如"120 80 60"）
- 实现方式：三个可拖动块（高压/低压/心率）
- 框宽度：
  - 高压框：固定3位数字宽度
  - 低压框：固定2位数字宽度
  - 心率框：固定2位数字宽度
- 操作流程：拖动→吸附→自适应宽度→提取
- 识别逻辑：覆盖到一个数字即识别相连所有数字
  - 例：低压110，框住1、0、10、11任一数字，都自动选择110
  - 覆盖后自动调整框宽度
